ApiController vs MVC controller :

Web API Controller : supplying data in various formats (XML, JSON, ...)
bv : public class ProductApiController : ControllerBase

MVC controller : can communicate with the Views (user interfaces) in an MVC program..
bv : public class ProductController : Controller -- >A controller would also work, but it creates an overhead of things we don't need.

[Route("api/controller")] --> route attribute is also mandatory
[ApiController]  --> We are also required to add this attribute
public class CourseController : ControllerBase

 [Route("api/[controller]")] vs [Route("api/controller")] vs [Route("api/Course")]:

 [Route("api/[controller]")] --> fills in the name of our controller automatically instead of just controller (so in the example above: Course).
 [Route("api/Course")] --> A hard-coded route is not a bad idea in itself, because changing the name of a controller must always be passed on to the clients using the API.
