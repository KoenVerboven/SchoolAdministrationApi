[12:44:00 WRN] No store type was specified for the decimal property 'TotalAmount' on entity type 'Invoice'.
This will cause values to be silently truncated if they do not fit in the default precision and scale.
Explicitly specify the SQL server column type that can accommodate all the values in 'OnModelCreating'
using 'HasColumnType', specify precision and scale using 'HasPrecision', or configure a value converter using 'HasConversion'.

Solution:

 public class Invoice
    {
        [Key]
        public int Id { get; set; }
        public required string InvoiceNumber { get; set; }
        public int StudentId { get; set; }
        
        [Column(TypeName = "money")]  --> ADD THIS : This line specifies the SQL Server column type for the TotalAmount property.
        public Decimal TotalAmount { get; set; }
     }

PS : the type from TotalAmount in SQL-Server : money

be careful :- The 'money' data type in SQL Server has a fixed precision of 19 digits and a scale of 4 decimal places. This means it can store values up to 922,337,203,685,477.5807. If you need to store larger values or more decimal places, you may want to consider using the 'decimal' data type with appropriate precision and scale instead.

https://learn.microsoft.com/en-us/sql/t-sql/data-types/money-and-smallmoney-transact-sql?view=sql-server-ver17
